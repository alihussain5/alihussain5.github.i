Feedback Bot

Kik Hack Week

Feedback Bot

Feedback Bot was created as a project for Kik Hack Week, a tri-annual hackathon held at Kik for its employees.

The bot asks users about their opinions on different features added to Kik, the data is collected and later processed into a summary.

A regular user is only allowed to leave feedback however if one is detected to be a Kik employee they can ask the bot for statistics. Asking the bot this will return the most recently processed summary of gathered data.

Technologies

Feedback Bot was written in Python built over an existing bot framework. In addition, it uses the NLTK library to process user input and Firebase to store and pull data.

Firebase was picked over a traditional back-end for its ease of use. In a hackathon, its important to save time where possible and Firebase does a fantastic job at that.

App Review Analysis

Side Project at Kik

App Review Analysis

During my work term at Kik I was tasked with finding a way to extract valuable information from app store reviews and parse it in an easily interpretable way. Kik receives hundreds of app store reviews a day with hundreds of thousands since release. Sifting through this data would be impossible by hand so we turn to implementing algorithms that do it for us. Being able to automatically categorize everything allows us to take this data to the next level and decide how to take advantage of it.

Break Down

Collecting Data

Review data was cyphoned from the Google Play store using their convenient gsutil and from the Apple App Store using a multitude of hacks involving scraping the iTunes Store, HTML/XML parsing, and for loops.

Parsing Data

Firstly, reviews are clustered based on categories. Topics included basic app functionality such as crashing, performance, bugs aswell as feature releases such as GIFs, media tray updates, and chat bubble colours. This was done using keyword detection and a hash table mapping keywords to categories. Using keywords was not the best idea due to typos and slang thus I was hoping to redo it using Google's Word2vec (word vectors), unfortunately I never got around to finishing the implentation.

Next up was deciphering between negative and positive sentiments. Each review has a rating, or stars, though after some researching and experimenting I concluded it was far from accurate. A sample of 3000 reviews were randomly selected from years ranging 2010 to 2015 and manually read through. Roughly 10% of these reviews had accurately used the rating system (this was judged by comparing the content of the review to its rating; specifically a lot of negative reviews still had 4-5 stars regardless of their content). This led to the decision of using natural language processing to properly classify this data.

The same 3000 reviews previously selected were manually classified between positive and negative and using NLTK, a Python natural language processing library, were used to train a sentiment classifier. This trained classifier was now used to classify each review on a scale of -1 to 1, replacing the stars system (-1 being equivalent to 0 stars, 0 to 3 and 1 to 5).

Presenting Data

Between update releases, each category would be assessed and compared to a 95% confidence interval to discover whether public opinion falls in the norm. This data, along with a few helpful things such as review count, were thrown into a web-based dashboard to be easily interpreted by users. For example, after 9.0 was released there was major backlash for Kik's decision to remove hashtag groups, the web dashboard would show an alert for "groups" as the wave of negative reviews were out of the norm.

Technologies

The program to collect and parse data was written Python. Collection was made possible by Google's gsutil on the Android side and BeautifulSoup, an HTML/XML parser, on the Apple side. Pandas' data frames were an integral part of managing the data and NLTK was used to take advantage of its Naive Bayes Classifier. MySQL was used to store the parsed data.

On the front end, JavaScript was used to show the interpreted data, a PHP script used to pull data from the MySQL database, and D3 to use that data within JavaScript. NVD3, a chart library for D3, was used to showcase fancy interactable graphs and present other useful information.

Study with Company

Hack the North

Study with Company

As a project for Hack the North, Study with Company was created to bring students with common courses together. Through a live campus map and checking in through web as well as Pebble watch, one can broadcast their location and course they're studying for.

Once broadcasted, anyone on campus studying for the same course can join them to create a study group.

Technologies

Web interface was created using JavaScript, jQuery, Google Maps API. Pebble app was created using JavaScript. Firebase was used instead of a back-end
